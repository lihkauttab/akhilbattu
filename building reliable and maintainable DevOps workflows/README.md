Over the recent period, I’ve been focusing on building reliable and maintainable DevOps workflows through hands-on implementation rather than tool-level exposure.
My work has involved designing and operating systems across the following areas:
• AWS Cloud – EC2, ECS (Fargate), S3, IAM, Route 53, CloudFront, CloudWatch, and CloudTrail, with emphasis on security, scalability, and observability
 • CI/CD – Implemented automated pipelines using GitLab CI/CD and Jenkins to standardize build and deployment processes
 • Containerization – Built and optimized Docker images and deployed containerized workloads to ECS
 • Infrastructure as Code – Provisioned and managed infrastructure using Terraform and AWS CloudFormation, ensuring repeatability and version control
 • Linux & Web Infrastructure – Worked extensively with Linux systems and Nginx, focusing on stability, performance, and troubleshooting
 • Migrations – Executed AWS Lightsail and full project migrations between accounts with minimal downtime and controlled rollouts
Key takeaways:
Simplicity in design reduces operational risk
Automation is valuable only when it is observable and debuggable
Strong monitoring and logging are non-negotiable in production systems
DevOps is less about tools and more about engineering discipline and ownership
Continuously improving systems, processes, and understanding — with a strong focus on reliability and long-term maintainability.
